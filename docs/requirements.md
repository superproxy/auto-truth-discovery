# 需求

## P0 — 当前主链路

1. **输入三分类落盘**：用户给一段散乱输入，skill 必须把它拆成需求/参考/测试三类，分别落到 `docs/requirements.md`、`docs/references.md`、`tests/scenarios/`，**不糊进 AGENTS.md**
2. **工程类型推断**：根据输入主语推断软件 / 方法论 / 文档 三种工程类型，给对应骨架
3. **AGENTS.md 高信号**：每行通过 "agent 没有这条会踩坑吗" 测试，仓库简单时 < 50 行
4. **dry-run 确认**：先输出方案预览（目录树），用户确认后再落盘

## P1 — 鲁棒性

5. 已有 `AGENTS.md` 时执行 reconcile（用 `edit` 逐段改），不整文件覆盖
6. 已有工程时只对**新增输入**做三分类，不重铺骨架
7. 命令必须从 `Makefile` / manifest / CI **实际找到**，找不到就标 TODO 而不是猜

## P2 — 体验

8. 提问只用一轮 `question` 工具，批量问完
9. 最小产出 5 项契约：AGENTS.md + 三分流文件 + 脚手架，缺一不可
10. 黑名单严格执行：不写通用建议、不抄需求细节、不留 placeholder

## 非目标

- 不做代码生成（只做工程骨架）
- 不做需求评审（只做分类）
- 不绑定具体语言/框架

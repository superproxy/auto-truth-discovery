# 问询记录

记录用户每次对本 skill 工程的问询类型，避免把提炼点、需求和参考资料混在一起。记录时使用正式、可追溯的表述，不直接照抄过于口语化的原句。

## 分类规则

| 类型 | 含义 | 落点 |
|---|---|---|
| 提炼点 | 对方法论、案例结论、表达方式的抽象总结 | 可进入 `SKILL.md`、案例或 checklist |
| 需求 | 对工程结构、行为规则、同步方式、记录方式的明确要求 | 需要改仓库文件或规则 |
| 参考资料 | 用户给出的外部材料、历史素材、理念来源、可借鉴对象 | 进入案例、references 或后续调研清单 |
